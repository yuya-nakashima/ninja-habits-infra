# AGENTS.md

このリポジトリ（`ninja-habits-infra`）で作業するエージェント向けのメモです。
AWS CDK TypeScript による IaC を扱います。

## 文書の読み方

- 共通方針とコーディングルールはワークスペース側 `../AGENTS.md` と `../docs/`（`../docs/rule-index.md` の「インフラ・デプロイ」行）から選ぶ
- このリポジトリ固有の詳細は `docs/` に置く

## 目的

- `ninja-habits-infra` の CDK スタックの実装・運用を進める
  - 依存順: network → database → api（network が VPC/SG/共有資源を持つ依存の根）、加えて独立な auth / hosting
- デプロイ方式、未決事項、判断の理由を失わないように残す

## 作業ルール

- デプロイ方式や運用の決定・未決事項は `docs/deploy.md` に追記する
- 複数リポジトリにまたがる方針・決定は、ワークスペース側 `../docs/decisions.md` に残す
- まだ決まっていないことは断定せず、候補として書く
- secrets（`.env` 実値、APIキー、DBパスワード、トークン）はコミットせず、AI にも読ませない

## ドキュメント配置

- デプロイ方式・運用メモ: `docs/deploy.md`
- 必要に応じてスタックごとの設計メモを `docs/` 配下へ追加する

## 記録フォーマット

以下のような形で簡潔に残すこと。種別は `決定` / `未決` / `提案` を使い分け、確定済みと未確定案を混ぜない。

```md
## YYYY-MM-DD

- 決定:        # 確定したこと（未決・提案の場合はこの行を置き換える）
- 理由:
- 次のアクション:  # 未決・提案で残課題があるとき
```
